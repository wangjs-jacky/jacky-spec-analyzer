# optimize-skill 迭代优化系统设计

> 创建日期: 2026-03-13
> 状态: 待实现

## 概述

设计一个迭代优化系统，用于优化 `spec-analyzer` skill 的提取能力。系统采用类似机器学习训练循环的方式，通过反复比对提取内容和目标内容来改进 skill 提示词。

## 核心理念

```
┌─────────────────────────────────────────────────────────────┐
│                    迭代优化循环                              │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  1. optimize-skill 读取目标内容（期望输出）                   │
│         ↓                                                    │
│  2. Agent 调用 spec-analyzer 提取 GitHub 仓库内容            │
│         ↓                                                    │
│  3. LLM 评估：对比提取内容 vs 目标内容                        │
│         ↓                                                    │
│  4. 生成差距报告 + 优化建议                                   │
│         ↓                                                    │
│  5. 应用优化到 spec-analyzer 提示词                          │
│         ↓                                                    │
│  6. 重复 2-5 直到满足评估阈值                                 │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

## 设计决策

| 决策点 | 选择 | 原因 |
|--------|------|------|
| 输出格式 | 无固定模板 | 灵活适应不同仓库，但需覆盖目标知识点 |
| 评估维度 | 知识点覆盖度 + 内容深度 | 用户明确需求 |
| 终止条件 | 达到评估阈值 | 自动化程度高 |
| 协作方式 | 子代理模式 + 文件记录 | A+C 混合方案 |

```

## 核心模块

### 模块 1：目标解析器 (Target Parser)

**功能**：解析目标内容，提取知识点清单

**输入**：
- 目标内容（Markdown 格式的期望输出）

**输出**：
- 知识点清单（每个知识点带权重和类型）

**示例输出**：

| ID | 知识点 | 权重 | 类型 |
|----|--------|------|------|
| K1 | AI 编程工作流的重要性 | 高 | 问题域 |
| K2 | Context Rot 问题 | 高 | 问题域 |
| K3 | GSD 概述 | 高 | 解决方案 |
| K4 | GSD 文件体系 | 高 | 解决方案 |
| K5 | 完整工作流程 | 高 | 解决方案 |
| K6 | Wave 执行机制 | 中 | 技术要点 |
| K7 | 里程碑功能 | 中 | 进阶功能 |
| K8 | 优缺点分析 | 中 | 评估 |

---

### 模块 2：评估器 (Evaluator)

**功能**：评估提取内容与目标内容的差距

**输入**：
- 目标知识点清单
- spec-analyzer 提取的内容

**评估维度**：

#### 1. 知识点覆盖度 (0-100%)

- **完全覆盖**：有详细解释，包含关键细节
- **部分覆盖**：有提及但不完整
- **未覆盖**：完全没有提到

#### 2. 内容深度 (1-5 分)

| 分数 | 描述 | 示例 |
|------|------|------|
| 1 | 仅提及 | "GSD 使用 Wave 执行" |
| 2 | 简要说明 | "Wave 执行是并行执行任务的方式" |
| 3 | 详细解释 | "Wave 执行将相同 way 编号的任务放入同一子代理串行执行..." |
| 4 | 包含示例 | 详细解释 + 代码/流程图示例 |
| 5 | 深入分析 | 包含原理、对比、权衡分析 |

**输出格式**：

```json
{
  "coverage": {
    "total": 15,
    "covered": 12,
    "partial": 2,
    "missing": 1,
    "percentage": 80
  },
  "depth": {
    "average": 3.2,
    "details": [
      {"id": "K1", "score": 4, "note": "详细解释了上下文过长问题"},
      {"id": "K2", "score": 2, "note": "只提到了文件，没有说明作用"}
    ]
  },
  "missing_points": ["K7: 里程碑功能"],
  "weak_points": ["K2: 文件体系说明不够详细"],
  "suggestions": [
    "在'扫描文档'步骤中添加对 PROJECT.md 等文件的说明",
    "添加'文件体系'专节，解释每个核心文件的作用"
  ]
}
```

---

### 模块 3：优化器 (Optimizer)

**功能**：根据评估结果生成 SKILL.md 修改建议

**输入**：
- 评估报告
- 当前 spec-analyzer SKILL.md

**输出**：
- 具体的修改建议
- 修改后的 SKILL.md

**优化策略**：

| 问题类型 | 优化策略 |
|----------|----------|
| 未覆盖知识点 | 添加新的提取规则或检查清单 |
| 深度不足 | 细化提取指令，添加深度要求 |
| 信息遗漏 | 添加必检项和提示词引导 |
| 格式问题 | 调整输出模板 |

## 工作流程详解

### 参数定义

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--target, -t` | 目标内容（文件路径或直接输入） | 必填 |
| `--repo, -r` | GitHub 仓库 URL | 必填 |
| `--threshold-coverage` | 覆盖度阈值 | 90% |
| `--threshold-depth` | 深度阈值 | 3.5 |
| `--max-iterations` | 最大迭代次数 | 5 |

### 执行步骤

#### 1. 初始化阶段

1. 读取目标内容
2. 调用目标解析器，生成知识点清单
3. 创建工作流目录 `./optimization-records/<timestamp>/`
4. 保存初始文件（target.md, repo-url.txt）

#### 2. 迭代循环

```python
for iteration in range(1, max_iterations + 1):
    # 2.1 调用 spec-analyzer
    result = agent_call("/spec-analyzer", repo_url, depth="deep")

    # 2.2 保存提取结果
    save(f"iterations/{iteration:02d}/extracted.md", result)

    # 2.3 评估
    evaluation = evaluate(target_points, result)
    save(f"iterations/{iteration:02d}/evaluation.json", evaluation)

    # 2.4 检查终止条件
    if evaluation.coverage >= threshold_coverage and \
       evaluation.depth >= threshold_depth:
        break

    # 2.5 生成优化建议
    suggestions = generate_suggestions(evaluation, current_skill)

    # 2.6 应用优化
    updated_skill = apply_suggestions(current_skill, suggestions)
    save(f"iterations/{iteration:02d}/changes.md", suggestions)
    current_skill = updated_skill
```

#### 3. 输出报告

1. 生成最终评估分数
2. 汇总迭代历史
3. 生成 SKILL.md 变更对比
4. 保存最终版本

## 评估提示词模板

```markdown
# 评估任务

## 目标知识点清单
{知识点清单}

## 提取的内容
{spec-analyzer 输出}

## 评估要求

请逐项评估每个目标知识点的覆盖情况：

### 1. 知识点覆盖度
对于每个目标知识点，判断：
- **完全覆盖**：有详细解释，包含关键细节
- **部分覆盖**：有提及但不完整
- **未覆盖**：完全没有提到

### 2. 内容深度评分 (1-5)
- 1: 仅提及名称
- 2: 简要说明（1-2 句话）
- 3: 详细解释（有完整描述）
- 4: 包含示例（有代码/图表/流程）
- 5: 深入分析（有对比、权衡、原理解释）

## 输出格式

请按以下 JSON 格式输出评估结果：

```json
{
  "coverage": {
    "total": <总数>,
    "covered": <完全覆盖数>,
    "partial": <部分覆盖数>,
    "missing": <未覆盖数>,
    "percentage": <覆盖百分比>
  },
  "depth": {
    "average": <平均深度分数>,
    "details": [
      {
        "id": "<知识点ID>",
        "score": <1-5>,
        "note": "<简短说明>"
      }
    ]
  },
  "missing_points": ["<未覆盖的知识点列表>"],
  "weak_points": ["<深度不足的知识点列表>"],
  "suggestions": [
    "<针对 SKILL.md 的具体修改建议>"
  ]
}
```
```

## 优化建议生成模板

```markdown
# 优化建议生成任务

## 当前评估结果
{评估 JSON}

## 当前 SKILL.md 内容
{当前 skill 内容}

## 任务

根据评估结果，生成具体的 SKILL.md 修改建议。

### 修改建议格式

对于每个问题，提供：

1. **问题描述**：具体是什么问题
2. **问题位置**：SKILL.md 中的哪个章节/步骤
3. **当前内容**：当前的提示词
4. **建议修改**：修改后的提示词
5. **修改原因**：为什么这样修改能解决问题

### 示例

**问题**: K4（GSD 文件体系）未被提取

**位置**: "扫描文档" 章节

**当前内容**:
> 递归查找所有 `.md` 文件，重点关注 README.md 和 docs/ 目录

**建议修改**:
> 递归查找所有 `.md` 文件，重点关注：
> - `README.md` - 项目介绍
> - `docs/` 目录 - 文档
> - `PROJECT.md` - 项目描述和目标
> - `REQUIREMENTS.md` - 需求清单
> - `ROADMAP.md` - 路线图
> - `STATE.md` - 状态追踪
> - `get-shit-done/templates/` - 模板文件（了解输出格式）

**原因**: 原描述太笼统，没有指定 spec-driven 仓库特有的文件类型
```

## 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| LLM 评估不稳定 | 评估结果波动 | 多次评估取平均，设置容差范围 |
| 迭代不收敛 | 无限循环 | 设置最大迭代次数 |
| 修改破坏现有功能 | 回归 | 保留原始版本，支持回滚 |
| 上下文过长 | 性能下降 | 使用 Agent 隔离，定期压缩 |
