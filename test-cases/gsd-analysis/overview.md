# GSD (Get Shit Done) 概览

## 一句话简介

GSD 是一个轻量级但强大的 **元提示词、上下文工程和规范驱动开发系统**，解决 Claude 在长对话中因上下文窗口填充导致的质量下降问题。

## 核心价值

**解决 Context Rot（上下文腐化）问题** —— 让 Claude Code 在大型项目中保持稳定、可靠的输出质量。

> 随着对话进行，Claude 的上下文窗口逐渐被填满，导致：
> - 输出质量下降（"我会更简洁"）
> - 遗忘早期决策
> - 偏离原始需求
> - 产生不一致的代码
>
> GSD 通过 **新鲜上下文 + 原子任务 + 规范追踪** 解决这个问题。

## 目标用户

- **独立开发者**：不写代码，让 Claude 写
- **小型团队**：不需要企业级流程（Jira、Sprint、故事点）
- **追求效率的人**：想要描述需求 → 获得可靠实现

> "我不是 50 人的软件公司。我不想玩企业戏剧。我只是一个有创意的人，想用 Claude 构建可靠的东西。"

## 快速开始

```bash
# 一键安装
npx get-shit-done-cc@latest

# 验证安装
/gsd:help

# 开始新项目
/gsd:new-project
```

## 核心工作流

```
┌─────────────────────────────────────────────────────────────┐
│                     GSD 工作流                               │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  1. /gsd:new-project     →  提问 → 研究 → 需求 → 路线图      │
│         ↓                                                    │
│  2. /gsd:discuss-phase N →  捕获实现偏好                     │
│         ↓                                                    │
│  3. /gsd:plan-phase N    →  研究 → 计划 → 验证              │
│         ↓                                                    │
│  4. /gsd:execute-phase N →  Wave 并行执行 → 原子提交         │
│         ↓                                                    │
│  5. /gsd:verify-work N   →  用户验收测试                     │
│         ↓                                                    │
│  6. 重复 2-5 直到里程碑完成                                   │
│         ↓                                                    │
│  7. /gsd:complete-milestone → 归档 + 打标签                  │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

## 关键特性

| 特性 | 说明 |
|------|------|
| **Wave 执行** | 独立任务并行，依赖任务串行 |
| **200K 新鲜上下文** | 每个任务在独立上下文中执行 |
| **XML Prompt 格式** | 结构化任务描述，精确无歧义 |
| **需求追踪** | REQUIREMENTS.md 中每个需求带 ID |
| **原子提交** | 每个任务完成立即 commit |
| **多 Agent 编排** | Orchestrator + 专门 Agent 模式 |

## 支持的平台

- Claude Code（主要）
- OpenCode
- Gemini CLI
- Codex

## 适用场景

**适合：**
- 从零开始构建新功能
- 有明确愿景的独立开发者
- 需要可靠 AI 辅助编码

**不适合：**
- 需要企业级流程的团队
- 模糊不清的需求探索
- 需要大量人工干预的任务
